# DWR-NEXT: Продолжение цикла разработки

Используй скилл **dwr-next** (`.cursor/skills/dwr-next/SKILL.md`) для продолжения DWR-цикла с новыми данными.

**Контекст предыдущей задачи загружается автоматически** из `task/{последняя_сессия}/session.md` — вводить его вручную не нужно.

**Входные данные от пользователя** — то, что пользователь написал после `/dwr-next`:
- логи, ошибки, трассировки из запуска/тестирования
- дополнительные требования или правки
- опционально: номер сессии (`NNN` или `NNN_datetime`), если нужна не последняя

Если новых данных нет — спроси: «Что нужно исправить или какие ошибки/логи нужно передать?»

Действуй как оркестратор по инструкциям из скилла dwr-next:
1. Найди последнюю (или указанную) сессию в `task/`, прочитай `session.md` → получи previous_task и affected_files.
2. Запусти суб-агента /task-decomposer с контекстом из session.md + новыми данными → получи список задач на исправление.
3. Запусти суб-агента /task-worker со списком задач → получи код и резюме изменений.
4. Запусти суб-агента /work-reviewer → получи отчёт ревью.
5. При критичных/важных замечаниях (раунд < 3): повтори fix loop (decomposer → worker → reviewer).
6. Обновляй `session.md` после каждого шага. Новые артефакты пиши в ту же директорию сессии с суффиксом `_next{M}`.
