# DWRB: Цикл разработки с Problem Solving

Используй скилл **dwrb** (`.cursor/skills/dwrb/SKILL.md`) для запуска полного цикла разработки с этапом систематического решения проблем вместо декомпозитора.

**Цель/запрос пользователя** — то, что пользователь написал после команды `/dwrb` (или в следующем сообщении). Если цели нет, спроси: «Какую задачу выполнить по циклу dwrb?»

Действуй как оркестратор по инструкциям из скилла dwrb:
1. Запусти **bmad-cis-problem-solving** с целью пользователя и `data=task.md` → получи `problem-solution-{date}.md`. На чекпоинтах доступны [a] Elicitation, [c] Continue, [p] Party-Mode, [y] YOLO.
2. Извлеки подзадачи из Implementation Plan / Action Steps документа.
3. Запусти суб-агента /task-worker с подзадачами → получи код и резюме изменений.
4. Запусти суб-агента /work-reviewer с кодом и контекстом задач → получи отчёт ревью. Всегда дополнительно adversarial review (review-adversarial-general).
5. При критичных/важных замечаниях (раунд < 3): запусти /task-decomposer с отчётом ревью → задачи на исправление → воркеры → снова шаг 4. Иначе заверши цикл.
