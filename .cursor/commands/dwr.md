# DWR: Цикл разработки

Используй скилл **dwr** (`.cursor/skills/dwr/SKILL.md`) для запуска полного цикла разработки.

**Цель/запрос пользователя** — то, что пользователь написал после команды `/dwr` (или в следующем сообщении). Если цели нет, спроси: «Какую задачу выполнить по циклу?»

Действуй как оркестратор по инструкциям из скилла dwr:
1. Запусти суб-агента /task-decomposer в отдельном контексте с целью пользователя → получи список подзадач с критериями готовности.
2. Запусти суб-агента /task-worker в отдельном контексте со списком подзадач → получи код и резюме изменений.
3. Запусти суб-агента /work-reviewer в отдельном контексте с кодом и контекстом задач → получи отчёт ревью.
4. При критичных/важных замечаниях (раунд < 3): запусти /task-decomposer с отчётом ревью → получи задачи на исправление → запусти /task-worker с этими задачами → снова шаг 3. Иначе заверши цикл.
