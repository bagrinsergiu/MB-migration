# DWR: Адаптивный цикл разработки

Используй скилл **dwr** (`.cursor/skills/dwr/SKILL.md`) для запуска цикла разработки.

**Цель/запрос пользователя** — то, что пользователь написал после команды `/dwr` (или в следующем сообщении). Если цели нет, спроси: «Какую задачу выполнить по циклу?»

**Оркестратор не читает код, не анализирует файлы, не пишет реализацию.**
Если задача неполная или контекст неясен — передать декомпозеру как есть, не собирать информацию самостоятельно.
Допустимые действия: создать `session.md` и `task.md`, запустить суб-агентов, обновить `Status`/`Round`.

Действуй как оркестратор по инструкциям из скилла dwr:
1. Запусти суб-агента /task-decomposer → получи список подзадач и поле `## Complexity` (nano/lite/full).
2. **Routing Gate**: выбери ветку по `## Complexity`:
   - `nano` → 1 воркер (fast model) → завершить без reviewer
   - `lite` → воркеры (fast) → 1 reviewer (fast), max 1 fix round при CRITICAL
   - `full` → полный пайплайн с параллельными волнами, до 3 fix rounds
3. При критичных замечаниях в lite/full: запусти /task-decomposer с отчётом ревью → воркеры → снова ревью. Иначе заверши цикл.
